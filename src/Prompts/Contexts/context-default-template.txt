## Components:
1. Search Input field:
- Keywords in testcase: search field, search bar, Enter a destination or property...etc
- Locator references: data-selenium="textInput"
- Function: to input a destination, hotel, address...etc that user want to search
- Keynotes: 
    + On click(No text) it will open an auto complete Panel (data-selenium="autocompletePanel") that displays hot domestic and international destinations
    + When being filled with text it will also open an auto complete Panel but contains a list of suggestions (data-testid="autosuggest-item")
    + When auto complete Panel displaying, must select options in it before performing any interactions on occupancy Box, checkin and checkout box, add flight and search button

2. Checkin and checkout Box:
- Keywords in test case: check in, check out, date, calendar...etc
- Locator references: data-element-name="check-in-box", data-element-name="check-out-box"
- Function: to select checkin and checkout date, and displaying selected date (dd/mmm/yyy)
- Keynotes:
    + On click will open checkin or checkout date picker component (data-selenium="rangePickerCheckIn")

3. Checkin or checkout date picker:
- Keywords in test case: date picker, next month, previous month...etc
- Locator references: data-selenium="rangePickerCheckIn", data-selenium="rangePickerCheckOut"
- Keynotes:
    + Will be opened when users click on Checkin/checkout Box, or after users select an auto complete Panel
    + Must refer current date when deal with test steps require to select date in tomorrow, next week, next 2 days, this weekend...etc
    + By default calendar tab will have 2 months for users to pick checkin/checkout date
    + If test steps mention any date that far ahead then 2 months please click ">" button (data-selenium="calendar-next-month-button") until see that date
    + When Checkin or checkout date picker component display, Must finish choose dates before performing any interactions on occupancy Box, add flight and search button
